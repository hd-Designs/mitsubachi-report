考えられる原因

結論: データ同期未完了（後追い補正される見込み）

以下の4つの証拠から、4/14の異常値は実際のビジネス悪化ではなく、データ取得パイプラインが前日分を取り切れていないことが原因であると判断する。

証拠1: ACL_売上の4/14行が極端に薄い

data/daily_kpi.xlsx の ACL_売上 シートを前日周辺で確認した結果:

4/13までは毎日50〜80件のコンスタントな注文があるが、4/14は新規2件・リピート0件と極端に薄い。リピートが完全にゼロという分布は統計的に極めて不自然（通常は新規:リピート ≒ 1:1）で、データ欠損を強く示唆する。

証拠2: Google Ads（google_naaa）の行がそもそも存在しない

広告結果集計 シートを前日周辺で確認した結果、google_naaa チャネルの値は 4/13 までは毎日計上されている（コスト 80,000〜150,000円台）が、4/14 だけ当該チャネルの行が丸ごと存在しない。FB/Yahoo は計上されているのに Google だけ欠落しているという分布は、広告媒体側の一時停止ではなく GAS 取得処理が Google Ads API から前日分を取り切れていない典型パターンである。

証拠3: Meta広告のCV(28)と実売上件数(2)の極端な乖離

kpi_snapshot.json の meta_ads_cv = 28 に対して、ACL_売上 での 4/14 実売上は 2 件のみ。通常の後追いCV乖離（±20%程度）を遥かに超えており、広告側はほぼ通常通りデータが届いているのに対し、売上データだけが取り込まれていないという解釈と整合する。

証拠4: AOVが異常値

AOV が ¥17,042（4週平均 ¥8,733 に対して +95.1%）。これは「売上全体が落ちた」のではなく、ごく一部の高単価注文しかデータに乗っていないことを示す。全件揃っていれば AOV は平均付近に回帰するはず。

曜日要因・季節要因

  • 火曜日は平均より売上がやや高い曜日で、急落する理由はない。
  • 4月中旬は採蜜シーズン序盤かつ新蜜予約の告知期で、需要が落ちる局面ではない（むしろ押し上げ要因）。
  • 4/13（月）までの推移は健全で、トレンドブレークは観測されていない。

補足所見（実害なし）

  • subscription_cancels = 2（+55.6%）は絶対値が小さく、4週平均1.3からの日次ブレ範囲内。警戒不要。
  • merchant_center_disapprovals = 155 は 4週平均 155前後で横ばい。既存の商品フィード不承認に変化なし。
  • mc_oos_rate = 29.5% は平均 40% より改善。要経過観察レベル。
